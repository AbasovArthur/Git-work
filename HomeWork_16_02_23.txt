## 1. Как скопировать на свой компьютер репозиторий из github.com,
что должно быть сделано предварительно?

кратко: “git clone *** “ ;
где *** : это ссылка(URL)
на клонируемый репозиторий.

• Вам нужно определить у себя на компе (локально) директорию
куда будет помещён репозиторий из github.com.
Открываем расположение локальной директории
в ком.строке “Git Bash” (подразумевается что
соответств софт установлен заранее),
перемещаться по папкам командами “cd *” и “cd.. *
где * - это название папки

• Репозиторий из github.com не копируется,
тут эта процедура называется клонированием

• Вводим команду: “git clone *** “ ;
где *** : это ссылка(URL) на клонируемый репозиторий.

• После ввода команды - в локальная директория
содержит клонированный репозиторий

## 2. Как создать git репозиторий из локального каталога на своем компьютере.
Как его отправить на github.com?

кратко:
Заливаем новый проект на Github:
git init
git add .
git commit -m "first commit"
git remote add origin ***.git
( “ *** “ : URL на репозиторий )
git push -u origin master

• Вам нужно создать на компе директорию,
которую впоследствии вы можете гордо
называть “локальный репозиторий”. Важно,
чтобы в этом репозитории был какой либо файл,
или папка.
Иначе нечему будет заливаться (пушиться) в github.com 

• ввести в ком.строке “Git Bash” команду : “git init”
- эта команда вводится один раз
- создает папку “.git” - в которой отслеживаются
дальнейшие манипуляции и изменения
"оперируемой" директории(репозитория)

• ввести в ком.строке “Git Bash” команду : “git add .”
в этом случае точка (“ . “) “зарядит” все файлы
в приближающийся коммит
• Совершить коммит:
ввести в ком.строке “Git Bash” команду : “git commit -m ‘ *** ‘ ”
где *** послужит названием коммита,
напр. - "first commit"

• далее командой : “git remote add origin ***.git ”
связываете локальный репозиторий с новым
репозиторием на  github.com
где *** : это ссылка (URL) в созданный репозиторий на сайте github.com.
актуально в случае когда на этом сайте
создан новый репо (github.com —> New —> Create a new repository —> …)

• команда “git push -u origin master”
запушит (отправит) все коммиты (изменения)
из локального репозитория в удаленный репозиторий на github.com
и свяжет их веткой "master" локального репозитория
с веткой "master" на удаленном репозитории.

## 3. Как получить на локальный компьютер изменения с github.com ?

кратко:
“git pull”

• либо удалить локальное содержимое директории,
вместе с скрытым файлом “.git”
и проделать процедуру ответа №1 (“git clone *** “)

• либо перейти в “оперируеммую” локальн. директорию,
знать что локальных изменений не было.
Если изменения могли быть или были,
то ввести команды:
“git add” - это внесет “под капод” ваши изменения
(в индекс .git);
и команду:
“git commit” чтоб “закоммитить” эти изменения
(тут ХтоЗнает.. но так надо)
все это это зафиксирует изменения в истории репозитория

• далее ввести команду “git pull”
команда обновит совершенные изменения
в репозитории github.com и зальет их в локальный репозиторий.

## 4. Какую информацию нам дают команда : git status git remote -v

• “git status” : отслеживает изменения и процесс коммита.

выводит в консоль совершенные изменения в репозитории.

показывает, какие файлы были изменены,
добавлены или удалены в вашей локальной директории
с тех пор, как вы последний раз выполняли коммит.

показывает информацию о текущей ветке
(будь это “master” && “main” …)

показывает файлы которые были
проиндексированы (“git add”) и готовы
к коммиту(“git commit -m” && use "git add" and/or "git commit -a")

• “git remote -v” : команда используется для отображения
имени и URL удаленного репозитория Git, со связанным с локальным репозиторием

ключ "-v" используется для отображения URL удаленного репозитория.

origin  git@github.com:AbasovArthur/Git-work.git (fetch)
origin  git@github.com:AbasovArthur/Git-work.git (push)
